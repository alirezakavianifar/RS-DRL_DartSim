MAPE-K Integration with RS-DRL Training: High-Level Flow

Overview

This document explains how the MAPE-K (Monitor-Analyze-Plan-Execute-Knowledge) architecture integrates with RS-DRL training to enable on-demand retraining when performance thresholds are violated.

Detailed Component Interactions

1. Training Loop Integration

Standard RL Loop (with MAPE-K hook):

obs, info = env.reset()
while timestep < total_timesteps:
    # Standard RL steps
    action, _ = model.predict(obs, deterministic=False)
    obs, reward, terminated, truncated, info = env.step(action)
    
    # MAPE-K Integration (every step)
    adaptation_result = mapek.step(timestep, reward, info)
    
    # If adaptation occurred (retraining)
    if adaptation_result and adaptation_result.get("executed"):
        # Reload updated model
        model = RSDRLDQN.load(adaptation_result["new_model_path"], env=env)
    
    # Continue training
    timestep += 1

2. MAPE-K Component Responsibilities

Monitor (Data Collection)

  • Input: timestep, reward, info from environment
  • Process: Extract metrics from environment step
  • Output: SystemMetrics object
  • Storage: Append to knowledge.metrics_history

Metrics Collected:

  • reward: Current reward
  • mission_success: Boolean
  • targets_detected: Count
  • team_destroyed: Boolean
  • decision_time: Milliseconds
  • episode_length: Steps

Analyzer (Violation Detection)

  • Input: SystemMetrics from Monitor
  • Process: Compare against thresholds
  • Output: AdaptationTrigger or None

Threshold Checks:

if reward < thresholds.min_reward:              # Default: 0.7
    violation_count++

if success_rate < thresholds.min_success_rate:  # Default: 0.8
    violation_count++

if decision_time > thresholds.max_decision_time:  # Default: 1000ms
    violation_count++

# Trigger if:
#   - violation_count >= retrain_threshold_violations (default: 3)
#   - AND timestep - last_retrain >= retrain_interval (default: 1000)

Planner (Adaptation Strategy)

  • Input: AdaptationTrigger, SystemMetrics
  • Process: Create adaptation plan
  • Output: AdaptationPlan dictionary

Plan Structure:

{
    "trigger": "threshold_violation",
    "action": "retrain",
    "timestep": 5432,
    "retrain_config": {
        "total_timesteps": 5000,
        "rho": 0.3,
        "learning_rate": 1e-4,
        "continue_training": True  # Continue from current model
    }
}

Executor (Apply Adaptation)

  • Input: AdaptationPlan, model_path, env
  • Process: Execute retraining
  • Output: ExecutionResult

Execution Steps:

  1. Load current model: RSDRLDQN.load(model_path, env=env)
  2. Continue training: model.learn(total_timesteps=5000)
  3. Save new model: model.save(new_model_path)
  4. Update Knowledge: Store new model path and history
  5. Return result with new_model_path

Knowledge (Shared Repository)

  • Stores:
    • Current model path
    • Metrics history (deque, maxlen=1000)
    • Model history (all retrained models)
    • Thresholds configuration
    • Adaptation history
  • Configurable:
    • retrain_threshold_violations: 3 (default)
    • retrain_interval: 1000 timesteps (default)

Trigger Conditions

Adaptation is triggered when ALL conditions are met:

  1. ✅ Threshold violation detected (reward, success rate, or decision time)
  2. ✅ Violation count >= retrain_threshold_violations (default: 3)
  3. ✅ Time since last retrain >= retrain_interval (default: 1000 steps)

Example Scenario:

Timestep 1000: Reward = -4.79 < 0.7 → violation_count = 1
Timestep 1500: Reward = -4.80 < 0.7 → violation_count = 2
Timestep 2000: Reward = -4.75 < 0.7 → violation_count = 3
                AND (2000 - 0) >= 1000 ✅
                → TRIGGER RETRAINING

Key Features

1. On-Demand Retraining

  • Only retrains when thresholds are violated
  • Efficient: avoids unnecessary retraining
  • Automatic: no manual intervention

2. Threshold-Based Adaptation

  • Configurable quality thresholds
  • Multiple metrics monitored
  • Violation counting prevents false triggers

3. Knowledge Persistence

  • Full history of adaptations
  • Metrics tracking
  • Model versioning
  • Save/load functionality

4. Seamless Integration

  • Non-intrusive: hooks into standard RL loop
  • Transparent: agent continues learning normally
  • Automatic model updates

Configuration Example

# Initialize Knowledge with thresholds
knowledge = Knowledge(
    thresholds=Thresholds(
        min_reward=0.7,                    # Minimum reward threshold
        min_mission_success_rate=0.8,      # Minimum success rate
        max_decision_time=1000.0           # Maximum decision time (ms)
    ),
    retrain_threshold_violations=3,        # Violations before retraining
    retrain_interval=1000                 # Min steps between retrains
)

# Create MAPE-K Manager
mapek = MAPEKManager(knowledge, env=env, model_path=initial_model_path)

Benefits of MAPE-K Integration

  1. Self-Adaptive: System adapts automatically to performance degradation
  2. Efficient: Only retrains when needed (on-demand)
  3. Robust: Handles changing conditions and drift
  4. Trackable: Full audit trail of adaptations
  5. Configurable: Adjustable thresholds and policies

Summary

The MAPE-K integration provides a self-adaptive RL system that:

  • Monitors performance in real-time
  • Detects threshold violations
  • Plans and executes on-demand retraining
  • Maintains knowledge of system state
  • Continues learning with updated models

This enables long-term autonomous operation with automatic adaptation to changing conditions and performance degradation.
