Overview

DARTSim (Dynamic Airspace Real-Time Simulation) is a mission simulator used for testing autonomous systems in a cyber-physical environment. We've wrapped it as a Gymnasium-compatible RL environment for training Deep Reinforcement Learning agents.

Key Characteristics

  • Domain: Autonomous team navigation in adversarial airspace
  • Goal: Complete missions (detect targets, avoid threats, survive)
  • Communication: TCP socket interface (port 5418)
  • RL Framework: Gymnasium (compatible with Stable-Baselines3)
  • Environment Type: Episodic, discrete action space, continuous state space

State Space

State Vector Structure

The state vector is a 19-dimensional normalized vector (with sensor_lookahead=5):

state_vector = [
    # Position (2 dims) - Normalized to [0, 1]
    position_x / map_size,    # [0, 1]
    position_y / map_size,    # [0, 1]
    
    # Direction (2 dims) - Normalized to [-1, 1]
    direction_x / max_dir,    # [-1, 1]
    direction_y / max_dir,    # [-1, 1]
    
    # Configuration (5 dims)
    altitude / max_altitude,  # [0, 1] - Current altitude level
    formation,                # 0.0 (LOOSE) or 1.0 (TIGHT)
    ecm,                      # 0.0 (OFF) or 1.0 (ON)
    ttc_inc_alt,              # [0, 1] - Time to complete increase altitude
    ttc_dec_alt,              # [0, 1] - Time to complete decrease altitude
    
    # Forward Threat Sensor (5 dims) - Boolean array
    threat_sensor[0],         # 0.0 or 1.0 - Threat in cell 1 ahead
    threat_sensor[1],         # 0.0 or 1.0 - Threat in cell 2 ahead
    threat_sensor[2],         # 0.0 or 1.0 - Threat in cell 3 ahead
    threat_sensor[3],         # 0.0 or 1.0 - Threat in cell 4 ahead
    threat_sensor[4],         # 0.0 or 1.0 - Threat in cell 5 ahead
    
    # Forward Target Sensor (5 dims) - Boolean array
    target_sensor[0],         # 0.0 or 1.0 - Target in cell 1 ahead
    target_sensor[1],         # 0.0 or 1.0 - Target in cell 2 ahead
    target_sensor[2],         # 0.0 or 1.0 - Target in cell 3 ahead
    target_sensor[3],         # 0.0 or 1.0 - Target in cell 4 ahead
    target_sensor[4],         # 0.0 or 1.0 - Target in cell 5 ahead
]

Total Dimensions: 2 + 2 + 5 + (2 × sensor_lookahead) = 19 (with sensor_lookahead=5)

State Components Explained

1. Position (Indices 0-1)

  • Source: state_data["positionX"], state_data["positionY"]
  • Normalization: Divided by map_size (default: 40)
  • Range: [0, 1]
  • Purpose: Team's current location on the map

2. Direction (Indices 2-3)

  • Source: state_data["directionX"], state_data["directionY"]
  • Normalization: Clipped to [-1, 1]
  • Range: [-1, 1]
  • Purpose: Team's movement direction
  • Values: Typically -1, 0, or 1 (left/right, up/down, stationary)

3. Configuration (Indices 4-8)

  • Altitude (Index 4): Current altitude level / max_altitude (0-1)
  • Formation (Index 5): Binary, 1.0 = TIGHT, 0.0 = LOOSE
  • ECM (Index 6): Binary, 1.0 = ON, 0.0 = OFF
  • TTC Increase Alt (Index 7): Time to complete altitude increase (normalized)
  • TTC Decrease Alt (Index 8): Time to complete altitude decrease (normalized)

4. Sensor Readings (Indices 9-18)

  • Threat Sensor (Indices 9-13): Boolean array for threats in forward 5 cells
  • Target Sensor (Indices 14-18): Boolean array for targets in forward 5 cells
  • Lookahead: Configurable via sensor_lookahead parameter (default: 5)

State Extraction Code

def _extract_state_vector(self, state_data: Dict[str, Any], 
                         threats: list, targets: list) -> np.ndarray:
    """Extract and normalize state vector from DARTSim state."""
    
    # Position (normalized by map size)
    pos_x = state_data.get("positionX", 0) / self.map_size
    pos_y = state_data.get("positionY", 0) / self.map_size
    
    # Direction (normalized to [-1, 1])
    dir_x = state_data.get("directionX", 0)
    dir_y = state_data.get("directionY", 0)
    dir_x_norm = np.clip(dir_x / max_dir, -1.0, 1.0)
    dir_y_norm = np.clip(dir_y / max_dir, -1.0, 1.0)
    
    # Configuration
    altitude = state_data.get("altitudeLevel", 0) / self.max_altitude
    formation = 1.0 if state_data.get("formation", 0) == 1 else 0.0
    ecm = 1.0 if state_data.get("ecm", False) else 0.0
    ttc_inc = state_data.get("ttcIncAlt", 0) / self.max_altitude
    ttc_dec = state_data.get("ttcDecAlt", 0) / self.max_altitude
    
    # Sensor readings (pad to sensor_lookahead length)
    threat_array = np.array([float(x) for x in threats[:self.sensor_lookahead]], dtype=np.float32)
    target_array = np.array([float(x) for x in targets[:self.sensor_lookahead]], dtype=np.float32)
    
    # Assemble state vector
    return np.array([
        pos_x, pos_y,
        dir_x_norm, dir_y_norm,
        altitude, formation, ecm,
        ttc_inc, ttc_dec,
        *threat_array,
        *target_array
    ], dtype=np.float32)

Action Execution

def _step_simulator(self, action: int, decision_time: float = 0.0) -> bool:
    """Execute action in simulator."""
    tactic = self.ACTIONS[action]  # Map ID to tactic string
    command = f'step ["{tactic}"] {decision_time}'
    response = self._send_command(command)
    return json.loads(response)  # Returns True if successful

Action Constraints

  • Altitude: Cannot go below 0 or above max_altitude (handled by DARTSim)
  • Formation: Binary switch (TIGHT ↔ LOOSE)
  • ECM: Binary switch (ON ↔ OFF)
  • Invalid Actions: DARTSim handles gracefully (may ignore or penalize)

Reward Function

Reward Type: Sparse + Multi-Objective

The reward function has two parts:

1. Step Reward (During Episode)

if not done:
    reward = -0.01  # Sparse step penalty
  • Purpose: Encourage faster mission completion
  • Value: -0.01 per step
  • Accumulation: Over episode length
  • Example: 500 steps = -5.0

2. Terminal Reward (Episode End)

Multi-objective weighted reward:

reward = 0.0
w = reward_weights

# Component 1: Mission Success
if mission_success:
    reward += w["w_success"] * 1.0  # +0.4

# Component 2: Targets Detected
targets_norm = min(targets_detected / max_targets, 1.0)
reward += w["w_targets"] * targets_norm  # +0.3 * normalized

# Component 3: Team Survival
if destroyed:
    reward += w["w_destruction"] * (-1.0)  # -0.5
else:
    reward += w["w_survival"] * 1.0  # +0.2

# Component 4: Decision Efficiency
decision_time_norm = 1.0 - min(decision_time / max_decision_time, 1.0)
reward += w["w_efficiency"] * decision_time_norm  # +0.1 * normalized

# Final: Clipped to [-1, 1]
reward = np.clip(reward, -1.0, 1.0)

Reward Weights (Default)

reward_weights = {
    "w_success": 0.4,      # Mission success (highest priority)
    "w_targets": 0.3,      # Targets detected
    "w_survival": 0.2,     # Team survival bonus
    "w_efficiency": 0.1,  # Decision time efficiency
    "w_destruction": 0.5   # Penalty for destruction
}

Reward Characteristics

  • Sparse: Most reward signal at episode end
  • Multi-Objective: Balances multiple goals
  • Bounded: Clipped to [-1, 1] for stability
  • Weighted: Prioritizes mission success

Example Reward Calculation

Scenario: Partial success (current agent behavior)

  • Steps: 500 (truncated)
  • Mission success: ✗
  • Targets detected: 2/10
  • Team survival: ✓
  • Decision time: 500ms (moderate)
Step penalty: 500 × (-0.01) = -5.0
Terminal:
  Mission success:  0.0
  Targets:          +0.3 × (2/10) = +0.06
  Survival:         +0.2
  Efficiency:       +0.1 × 0.5 = +0.05
  Total:            +0.31

Final: -5.0 + 0.31 = -4.69 ≈ -4.79 ✓

Episode Management

Episode Termination Conditions

  1. Natural Termination (terminated=True):

    • Mission completed successfully
    • Team destroyed
    • Checked via: finished() TCP command

  2. Truncation (truncated=True):

    • Maximum episode steps reached
    • Default: 1000 steps (training), 500 steps (evaluation)
    • Prevents infinite episodes

Episode Lifecycle

reset() → initial_state
    ↓
step(action) → state, reward, done, info
    ↓
step(action) → state, reward, done, info
    ↓
... (repeat until done or truncated)
    ↓
final_state, terminal_reward
    ↓
reset() → new episode

Reset Process

def reset(self, seed=None, options=None):
    """Reset environment to initial state."""
    # 1. Reconnect to DARTSim if needed
    if not self.connected:
        self._connect()
    
    # 2. Get initial state
    state_data = self._get_state()
    threats = self._read_sensor("Threat", self.sensor_lookahead)
    targets = self._read_sensor("Target", self.sensor_lookahead)
    
    # 3. Extract state vector
    observation = self._extract_state_vector(state_data, threats, targets)
    
    # 4. Reset episode counter
    self.episode_step = 0
    
    return observation, info

Step Process

def step(self, action: int):
    """Execute one step in the environment."""
    # 1. Execute action in simulator
    self._step_simulator(action)
    
    # 2. Check if finished
    done = self._finished()
    
    # 3. Get updated state
    observation = self._get_updated_state()
    
    # 4. Compute reward
    if done or truncated:
        results = self._get_results()
        reward = self._compute_reward(results, done=True)
    else:
        reward = self._compute_reward({}, done=False)
    
    # 5. Check truncation
    truncated = (self.episode_step >= self.max_episode_steps)
    
    return observation, reward, terminated, truncated, info

TCP Interface

Connection Details

  • Protocol: TCP socket
  • Host: localhost (default)
  • Port: 5418
  • Timeout: 30 seconds (connection), 10 seconds (command)

TCP Commands

Command Description Returns
getState Get current team state JSON dict
readForwardThreatSensor N Read threat sensor (N cells) Boolean array (JSON)
readForwardTargetSensor N Read target sensor (N cells) Boolean array (JSON)
finished Check if simulation finished Boolean (JSON)
step ["tactic"] time Execute action Boolean (JSON)
getResults Get mission results JSON dict

Connection Management

def _connect(self) -> bool:
    """Connect to DARTSim TCP server."""
    self.socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    self.socket.settimeout(30.0)
    self.socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    self.socket.connect((self.host, self.port))
    self.connected = True

Features:

  • Automatic reconnection on failure
  • Timeout handling
  • Error recovery

Implementation Details

Key Configuration Parameters

DARTSimEnv(
    host="localhost",              # DARTSim TCP host
    port=5418,                     # DARTSim TCP port
    sensor_lookahead=5,            # Forward sensor cells
    map_size=40,                   # Max map size (normalization)
    max_altitude=4,                # Max altitude level
    max_targets=10,                # Max targets (reward normalization)

)

Error Handling

  • Connection Loss: Automatic reconnection
  • Timeout: 30s connection, 10s per command
  • Invalid State: Returns zero vector if state fetch fails
  • Action Failures: Retries with reconnection

Performance Considerations

  • TCP Latency: Each step requires network round-trip
  • State Extraction: Minimal computation (normalization)
  • Reward Computation: Computed at episode end (efficient)
  • Episode Length: Truncation prevents infinite loops

Usage Example

from dartsim_env import DARTSimEnv

# Create environment
env = DARTSimEnv(
    max_episode_steps=1000,
    sensor_lookahead=5,
    verbose=False
)

# Reset
obs, info = env.reset()

# Run episode
for step in range(1000):
    # Select action (e.g., from agent)
    action = env.action_space.sample()  # Random action
    
    # Step
    obs, reward, terminated, truncated, info = env.step(action)
    
    # Check if done
    if terminated or truncated:
        print(f"Episode ended: reward={reward}")
        obs, info = env.reset()

# Close
env.close()

MAPE-K Integration

Overview

The DARTSim environment integrates with MAPE-K (Monitor-Analyze-Plan-Execute-Knowledge) architecture for self-adaptive systems, enabling on-demand retraining when performance thresholds are violated.

Integration Flow

See MAPEK_INTEGRATION_FLOW.md for detailed high-level flow diagram and component interactions.

Key Points:

  • MAPE-K monitors performance metrics every training step
  • Analyzer detects threshold violations (reward, success rate, decision time)
  • Planner creates retraining plans when violations detected
  • Executor performs on-demand model retraining
  • Knowledge repository maintains adaptation history

Benefits:

  • Self-adaptive: Automatically adapts to performance degradation
  • Efficient: Only retrains when needed
  • Robust: Handles changing conditions

Key Design Choices

  1. Sparse Rewards: Encourages learning long-term strategies
  2. Normalized States: Stable for neural network training
  3. Discrete Actions: Simpler than continuous control
  4. Multi-Objective: Balances mission goals
  5. Episode Truncation: Prevents infinite episodes
  6. MAPE-K Integration: Enables self-adaptive retraining

This environment is ready for RL training with algorithms like DQN, PPO, or any Gymnasium-compatible method, with optional MAPE-K integration for autonomous adaptation!
